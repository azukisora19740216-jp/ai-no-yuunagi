# ADR-0004: 実証運用、招待、利用者資格、同意、KYC

- 状態: 採用（PD-01〜03基盤実装、段階有効化・本番運用値は未承認）
- 決定日: 2026-07-21
- 対象: 実証地域、登録制御、招待、年齢・アカウント、規約同意、KYC、受渡情報
- 置換対象: ADR-0002のKYC・会員条件に関する暫定判断

## 背景

制度資料受領後の差分レビューで、対象地域、招待方式、登録上限、年齢、法人利用、KYC必須操作、受渡情報の開示時点が本番判断待ちとして残った。2026-07-21にプロダクト責任者が以下を正式仕様として決定した。本ADRは法的評価を行うものではない。

## 決定

### 実証運用

- 対象地域は岡山県倉敷市および周辺地域とする。
- 初期運用は招待制とする。
- 登録上限は設定値で管理し、初期値を50名とする。
- 全国公開機能はfeature flagで無効にする。未設定時も無効とする。
- 受渡しは対面手渡しを原則とする。配送は別途定めた条件で提供者が任意に選ぶ場合だけ扱う。
- 「周辺地域」の具体的な市区町村または郵便番号範囲は運用設定として承認前に確定する。コードへ推測で埋め込まない。

### 招待

- 招待コードは運営者が発行する単回利用コードとする。
- 初期段階では一般利用者による自由な再招待機能を提供しない。
- 招待元、発行者、発行日時、使用日時、失効日時、使用者を追跡できるようにする。
- 生の招待コードは保存せず、検証用ハッシュを保存する。発行時だけ安全な経路で提示する。
- コード使用と会員枠の確保は同一DBトランザクションで排他的に処理し、二重使用および上限超過を拒否する。
- 招待の発行、失効、使用成功・拒否を理由付き監査ログへ記録する。監査ログへ生コードを残さない。

### 年齢・アカウント・同意

- 利用者は18歳以上の個人に限定する。
- 1人1アカウントとする。
- 法人・団体アカウントは初期MVPで受け付けない。
- 登録時に利用規約同意、プライバシーポリシー同意、18歳以上であることの確認を、それぞれ文書版・確認時刻・確認経路とともに追記記録する。
- 規約・ポリシー改定時の再同意要否は文書版ごとの設定で扱う。既存の同意記録を更新して上書きしない。
- 1人1アカウントの確認手段は、必要最小限の個人情報とKYC結果を用いる方針とする。具体的な照合キー、保持期間、例外審査はプライバシー・運用レビュー後に決める。

### KYCと受渡情報

- 公開情報の閲覧と仮登録にはKYCを要求しない。
- 出品、受取申込み、取引参加コマンドには、その時点で有効な `verified` KYCを必須とする。
- 「取引参加」には、受取人承諾、受渡日程・方法の合意、取引メッセージ、完了報告を含む。KYC失効・取消し後の進行中取引は自動取消しにせず管理レビューへ送る。
- 正確な受渡情報は、受取人が承諾して取引が成立した後、その取引の提供者と受取人だけに必要最小限開示する。管理者アクセスは業務上必要な権限と監査を要する。
- 本番KYC導入前は差し替え可能なアダプター、feature flag、開発モックを使う。
- 本番環境でモックKYCが有効、または本番KYC設定が欠落した場合は、KYC必須操作をfail closedで拒否する。
- 本人確認書類の実画像をseed、fixture、通常ログへ保存しない。

## 設定値

少なくとも次をサーバー側の型付き設定として持ち、クライアント値を信頼しない。

| 設定                           |          初期値 | 未設定時             |
| ------------------------------ | --------------: | -------------------- |
| `PILOT_MODE_ENABLED`           |          `true` | 有効                 |
| `PILOT_REGISTRATION_LIMIT`     |            `50` | 登録拒否             |
| `NATIONWIDE_PUBLIC_ENABLED`    |         `false` | 無効                 |
| `KYC_REQUIRED_FOR_TRANSACTING` |          `true` | 必須                 |
| `KYC_ADAPTER`                  | 開発のみ `mock` | 本番は必須操作を拒否 |

環境変数名は実装時に既存の設定規約へ合わせて確定してよいが、意味と安全側既定値は変更しない。

## 影響

- 現行のopen registration、KYCゲートなしの掲載・申込み・取引は本ADRと不一致である。
- 招待、同意、年齢確認、登録上限のデータ構造とサーバー側サービスが必要になる。
- KYC状態だけでなく有効期限・決定元・履歴を扱い、全コマンドで認可と同時に検証する必要がある。
- 地域範囲、登録上限の算入対象、1人1アカウントの照合方法は実装前の運用設定・プライバシー判断として残る。

## 非決定事項

- 「周辺地域」の具体範囲
- 登録上限50名に管理者・モデレーター・退会済み・開発用アカウントを含めるか
- 1人1アカウント確認に使う照合情報と例外手続
- 本番KYC事業者、KYC結果・参照情報の保持期間

これらを独断で固定せず、未指定の場合に資格要件を迂回できない設計とする。

## 実装追記（2026-07-21）

- expand migration `202607210004_policy_pilot_identity_expand` にsidecar型の制度版、pilot設定、招待、会員資格、同意、KYC履歴、本人重複claimを追加した。
- 招待コードは乱数生成後にSHA-256ハッシュだけを保存し、発行時に限り生コードを返す。使用と登録枠確保はadvisory lockを含む単一DBトランザクションで処理する。
- `FEATURE_PILOT_ENROLLMENT` と `FEATURE_KYC_GATES` を既定offで追加した。設定を有効にして承認済みDB設定がない場合は登録・取引操作を拒否する。
- KYCはadapter interfaceと開発mockを分離した。本番環境ではmockのサービス処理と管理フォームを無効にする。本番adapterは未実装である。
- 対象地域の実値、枠算入対象、本人照合キー・保持期間、招待期間は設定または運用判断のままであり、コードへ本番値を固定していない。
