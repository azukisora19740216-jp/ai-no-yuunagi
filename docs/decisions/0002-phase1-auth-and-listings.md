# ADR-0002: フェーズ1の認証・掲載・受取申込み

- 状態: 一部置換済み（ADR-0004）
- 日付: 2026-07-20
- 対象: 認証、会員、掲載、投稿審査、受取申込み

## 背景

> 2026-07-21追記: 会員条件、招待制、地域、登録上限、KYC必須操作、同意記録はADR-0004を正とする。本ADRの「KYCをゲートにしない」という暫定判断は置換された。実装はまだADR-0004へ追随していない。

要件文書との整合を確認した結果、フェーズ1の実装に影響する未決事項は `requirements-conflicts.md` の C-001、C-003、C-108、C-109、C-110、C-111、C-115、C-121 に残っている。予定されている利用規約、禁止品基準、配送運用基準等の原資料はまだ配置されていない。

以下は本番仕様の確定ではなく、安全側で開発と検証を進めるための暫定判断である。

## 決定

### 認証とメール確認

- 独自のパスワード処理を実装せず、Better Authのメール・パスワード認証を使用する。
- パスワードは認証用 `accounts` テーブルのハッシュだけを保存する。
- メール未確認の利用者にはセッションを発行しない。
- 開発環境では確認URLを `mock_emails` に保存し、ローカル専用画面で確認できるようにする。本番ではモック設定を起動時に拒否する。
- パスワードリセットと本番メール配信アダプターは次フェーズ以降とする。

### 会員・権限

- 新規登録者には `USER` ロールを付与する。
- フェーズ1の投稿承認は `MODERATOR` と `ADMINISTRATOR` に限定する。
- `DONATION_REVIEWER` は投稿承認不可、`AUDITOR` は閲覧専用とする。
- 詳細権限マトリクスが承認されるまでは、用途別の最小権限だけをコード上のポリシーとして持つ。
- KYC必須操作は未決のため、フェーズ1ではKYCを登録・掲載・申込みのゲートにしない。

### 物品と禁止品

- 正式な禁止品基準がないため、seedされた低リスク想定カテゴリーだけを暫定許可リストとして使用する。
- すべての投稿を人手審査し、自動公開しない。
- `DRAFT / REJECTED` だけを編集・再申請可能とし、`PENDING_REVIEW` だけを承認または差戻し可能とする。
- 金額、希望価格、希望ポイント、送料支払いのフィールドを設けない。
- `item_images` の構造は用意するが、実行形式判定・画像再エンコード・マルウェア検査が未実装の間はアップロードUIを提供しない。

### 受取申込みと選択

- C-108の推奨案に従い、申込みを `item_requests` として取引から分離する。
- 公開中の物品だけ申込み可能とし、提供者自身の申込みを禁止する。
- 同一物品・同一申込者の再申込みは暫定的に一意制約で禁止する。撤回後の再申込み可否は人間の決定待ち。
- 提供者が1件を選択した時点で、物品を `RESERVED`、選択申込みを `SELECTED`、他の未処理申込みを `NOT_SELECTED` にする。
- 受取人の承諾と `transactions` の生成は、C-106の決定後に実装する。

### 一貫性と監査

- 状態の汎用更新APIを作らず、下書き保存、投稿申請、審査、申込み、受取人選択のサービスコマンドを用意する。
- 受取人選択はDBトランザクション、楽観ロック、部分ユニーク制約を併用する。
- 会員登録、メール確認、プロフィール更新、掲載操作、申込み、受取人選択を監査対象とする。
- 監査差分にはメール、申込み本文、プロフィール本文を保存しない。

## 影響と見直し条件

禁止品基準、利用規約、KYC条件、ロール権限、受取人選択後の承諾フローが承認された時点で本ADRを見直す。法的適合性を示す決定ではない。
